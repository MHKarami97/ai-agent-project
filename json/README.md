# JSON Formatter و Validator

این پروژه یک رابط کاربری ساده برای فرمت‌دهی، اعتبارسنجی و نمایش درختی ساختارهای JSON در مرورگر است. تمام عملیات روی کاربر و توسط مرورگر انجام می‌شود، بنابراین برای میزبانی می‌توان از GitHub Pages یا هر هاست استاتیک دیگر استفاده کرد بدون نیاز به بک‌اند.

## ساختار پوشه‌ها

- `index.html` – صفحه اصلی با ساختار RTL و لینک به فایل‌های CSS و JS جداگانه.
- `assets/css/styles.css` – دریافت فونت `Vazirmatn` از Google Fonts، استایل‌های اصلی، حالت موبایل، و تنظیمات دکمه/ورودی.
- `assets/js/main.js` – مدیریت تبدیل JSON، ذخیره/بارگذاری از `localStorage`، ساخت درخت نمایش، نمونه‌های آماده، و وضعیت پیام‌ها.

## ویژگی‌ها

1. **فرمت‌دهی JSON**: با کلیک روی "فرمت کردن"، متن موجود در `textarea` به‌صورت منظم نمایش داده می‌شود.
2. **اعتبارسنجی ساختار**: بررسی می‌کند که متن وارد شده JSON معتبری است یا خیر و نتیجه را نمایش می‌دهد.
3. **نمایش درختی**: ساختار JSON در بخش درخت نمایش داده می‌شود و برای هر آبجکت یا آرایه جزئیات روشن است.
4. **ذخیره در مرورگر**: ورودی‌ با استفاده از `localStorage` حفظ می‌شود تا پس از بارگذاری مجدد، داده از دست نرود.
5. **نمونه‌های فوری**: دکمه‌هایی برای بارگذاری سریع JSON نمونه بدون تایپ دستی.
6. **ظاهر RTL و فونت Vazirmatn**: تمام متن‌ها، دکمه‌ها و جعبه‌های ورودی از فونت فارسی `Vazirmatn` استفاده می‌کنند.
7. **ریسپانسیو**: استفاده از گرید با حداقل عرض ۳۲۰ پیکسل برای نمایش مناسب در موبایل.

## نحوه استفاده

1. پروژه را در پوشه‌ای روی سیستم خود کپی کنید.
2. از هر مرورگر مدرن، فایل `index.html` را باز کنید یا پروژه را روی GitHub Pages قرار دهید.
3. JSON را در جعبه متن قرار دهید، سپس گزینه فرمت یا اعتبارسنجی را انتخاب کنید.
4. در صورت وجود JSON صحیح، نمایش درختی برای مشاهده ساختار فعال می‌شود.
5. داده‌ها به‌صورت خودکار در `localStorage` ذخیره می‌شوند و با بازگشت به صفحه، بازیابی خواهند شد.

## میزبانی روی GitHub Pages

1. تمام فایل‌ها را به یک مخزن git اضافه کنید.
2. شاخه‌ای برای GitHub Pages مانند `gh-pages` تعریف کنید.
3. تنظیمات مخزن را باز کنید، به بخش GitHub Pages بروید و شاخه‌ی مورد نظر را به عنوان منبع صفحه انتخاب کنید.
4. پس از چند دقیقه، سایت استاتیک آماده نمایش خواهد بود.

## تست‌ها

چون این پروژه مبتنی بر مرورگر است، برای تست می‌توان:

- `index.html` را در مرورگر باز کرد و JSON‌های مختلف را چک کرد.
- با ابزار DevTools بررسی کرد آیا هیچ خطایی در کنسول نمایش داده نمی‌شود.
- میزان سازگاری با مرورگرها و واکنش‌پذیری را با تغییر اندازه‌ی پنجره آزمایش کرد.

اگر قصد دارید فایل‌ها را تغییر دهید، فقط کافی‌ست فایل‌های داخل `assets` را ویرایش کرده و دوباره در مرورگر بارگذاری کنید.
