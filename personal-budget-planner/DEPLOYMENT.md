# 📘 راهنمای استقرار در GitHub Pages

این راهنما به شما کمک می‌کند تا برنامه مدیریت بودجه شخصی را در GitHub Pages مستقر کنید.

## 🚀 مراحل استقرار

### مرحله 1: ایجاد مخزن GitHub

1. به [GitHub.com](https://github.com) بروید و وارد حساب کاربری خود شوید
2. روی دکمه **New** (مخزن جدید) در گوشه بالا سمت راست کلیک کنید
3. نام مخزن را وارد کنید (مثلاً `personal-budget-planner`)
4. مخزن را **Public** قرار دهید
5. روی **Create repository** کلیک کنید

### مرحله 2: آپلود فایل‌ها

#### روش 1: آپلود مستقیم از طریق وب

1. در صفحه مخزن جدید، روی **uploading an existing file** کلیک کنید
2. تمام فایل‌های پروژه را بکشید و در صفحه رها کنید:
   - `index.html`
   - `style.css`
   - `script.js`
   - `README.md`
   - `.gitignore`
3. در کادر پایین توضیح Commit را بنویسید: "Initial commit"
4. روی **Commit changes** کلیک کنید

#### روش 2: استفاده از Git Command Line

```bash
# باز کردن پوشه پروژه در ترمینال
cd D:\Local\ai-agent-project\personal-budget-planner

# مقداردهی اولیه Git
git init

# افزودن تمام فایل‌ها
git add .

# ایجاد Commit
git commit -m "Initial commit"

# اضافه کردن مخزن GitHub به عنوان origin
git remote add origin https://github.com/USERNAME/personal-budget-planner.git

# Push کردن به GitHub
git branch -M main
git push -u origin main
```

**نکته**: `USERNAME` را با نام کاربری GitHub خود جایگزین کنید.

### مرحله 3: فعال‌سازی GitHub Pages

1. در صفحه مخزن، روی تب **Settings** کلیک کنید
2. در منوی سمت راست، روی **Pages** کلیک کنید
3. در بخش **Source**:
   - Branch: `main` را انتخاب کنید
   - Folder: `/ (root)` را انتخاب کنید
4. روی **Save** کلیک کنید
5. صبر کنید تا پیغام زیر ظاهر شود:
   ```
   Your site is published at https://USERNAME.github.io/personal-budget-planner/
   ```

### مرحله 4: دسترسی به سایت

بعد از 1-2 دقیقه، سایت شما در آدرس زیر در دسترس خواهد بود:

```
https://USERNAME.github.io/personal-budget-planner/
```

## ✅ تست و بررسی

بعد از استقرار، موارد زیر را بررسی کنید:

- [ ] صفحه به درستی بارگذاری می‌شود
- [ ] فونت فارسی به درستی نمایش داده می‌شود
- [ ] می‌توانید تراکنش اضافه کنید
- [ ] نمودارها به درستی کار می‌کنند
- [ ] ویرایش و حذف تراکنش کار می‌کند
- [ ] داده‌ها بعد از رفرش صفحه حفظ می‌شوند
- [ ] خروجی CSV به درستی کار می‌کند

## 🔄 به‌روزرسانی پروژه

برای به‌روزرسانی پروژه در GitHub Pages:

### روش 1: آپلود مستقیم

1. به صفحه فایل مورد نظر در GitHub بروید
2. روی آیکون مداد (Edit) کلیک کنید
3. تغییرات را اعمال کنید
4. روی **Commit changes** کلیک کنید

### روش 2: استفاده از Git

```bash
# ویرایش فایل‌ها
# سپس:

git add .
git commit -m "توضیحات تغییرات"
git push
```

بعد از Push، تغییرات به صورت خودکار در GitHub Pages اعمال می‌شوند (ممکن است 1-2 دقیقه طول بکشد).

## 🌐 دامنه سفارشی (اختیاری)

می‌توانید یک دامنه سفارشی به پروژه خود اضافه کنید:

1. یک دامنه تهیه کنید
2. در تنظیمات دامنه، یک CNAME record با مقدار `USERNAME.github.io` ایجاد کنید
3. در صفحه GitHub Pages Settings، دامنه خود را در قسمت **Custom domain** وارد کنید
4. **Enforce HTTPS** را فعال کنید

## 🔧 رفع مشکلات رایج

### صفحه 404 نمایش می‌دهد

- مطمئن شوید که فایل `index.html` در ریشه مخزن قرار دارد
- در تنظیمات Pages بررسی کنید که Branch درست انتخاب شده است
- 5-10 دقیقه صبر کنید (اولین بار ممکن است کمی طول بکشد)

### فونت‌ها نمایش داده نمی‌شوند

- مطمئن شوید که اتصال اینترنت برقرار است
- Cache مرورگر را پاک کنید (Ctrl + Shift + Delete)

### نمودارها کار نمی‌کنند

- Console مرورگر را باز کنید (F12)
- اگر خطای CORS مشاهده کردید، مطمئن شوید که Chart.js از CDN بارگذاری می‌شود
- حداقل یک تراکنش اضافه کنید

### تغییرات نمایش داده نمی‌شوند

- Cache مرورگر را پاک کنید
- از حالت Incognito/Private استفاده کنید
- 2-3 دقیقه صبر کنید تا GitHub Pages به‌روز شود
